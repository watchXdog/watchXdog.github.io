折腾看板娘

初入看板娘

一开始咱看到第一个链接有个看板娘作为二次元死宅肯定是要去搞一个的，方法挺简单的就是在博客跟目录下执行npm install -save hexo-helper-live2d 并且在站点配置文件下加入
live2d:
enable: true
scriptFrom: local
pluginRootPath: live2dw/
pluginJsPath: lib/
pluginModelPath: assets/
tagMode: false
model:
use: live2d-widget-model-wanko #选择哪种模型
display: #放置位置和大小
position: right
width: 150
height: 300
mobile:
show: false #是否在手机端显示
之后在live2d找到你喜欢的萌妹子，npm install live2d-widget-modelname 即可在你的博客中添加你喜欢的萌妹子

再入看板娘

看了一下live2d的git模型有点老想着去找点可爱一点的萌妹子，于是开始了快乐的寻纸片人老婆之路。
最后终于找到了一个新的模型库地址
不过这个模型库的使用方法跟上面有点不一样 方法如下：

在博客根目录下创建一个live2d_models文件夹.
在此文件夹内新建一个子文件夹.
将你的 Live2D 模型复制到这个子文件夹中.
将子文件夹的名称输入_config.yml的model.use中. 　

这样你就能使用更多的萌妹子了！！！

##三入看板娘
无聊百度看到了一个大佬博客的看板娘能说话和换衣服还能（唱跳rap篮球）换人 瞬间羡慕 之后百度了一会找到了安装的教程。
接下来就开始躺坑了。
首先跟着大佬步骤走在git上下好文件
1.之后解压到本地博客目录的themes/next/source下，修改autoload.js文件，如下：
const live2d_path = “https://cdn.jsdelivr.net/gh/stevenjoezhang/live2d-widget/"; 改为const live2d_path = “/live2d-widget/“;
2.在/themes/next/layout/_layout.swing中,新增如下内容：
< script src=”/live2d-widget/autoload.js”>< /script>(前面的空格记得删)
3.在 主题配置文件 中,新增如下内容：
live2d:
enable: true
4.想修改看板娘大小、位置、格式、文本内容等，可查看并修改 waifu-tips.js 、 waifu-tips.json 和 waifu.css。

这样就能看到看板娘（个屁）
这样你跟着做后发现怎么看板娘没了，之后我慢慢摸索百度了x个小时后还是不行遂暴毙离去。
第二天重新浏览了原git网址 发现居然还要在.在/themes/next/layout/_layout.swing中的< head>中加入
< script src=”https://cdn.jsdelivr.net/npm/jquery/dist/jquery.min.js">< /script>
< link rel=”stylesheet” href=”https://cdn.jsdelivr.net/npm/font-awesome/css/font-awesome.min.css"/>
来启用jQuery和font-awesome插件
之后在加入< script src=”https://cdn.jsdelivr.net/gh/stevenjoezhang/live2d-widget/autoload.js">< /script>

看板娘就出来了大功告成
然而事情还没有结束 如果你在看到这里之前安装过 hexo-helper-live2d 你会发现你有两个看板娘哎。这里你如果想只有一个就使用npm uninstall -save –dev hexo-helper-live2d 这样就只有一个看板娘了。（如果你要两当我没说）

最后一个坑就是 手贱点了看板娘的x后看板娘不见了 刷新也不见了。怎么搞都不出来 最后只有清除缓存才行。
ps:上面问题解决了，找到了作者的git 在issue留了言 作者把功能更新出来了 现在可以点x后恢复看板娘了！！！

溜了溜了 明天看情况更新 开始学习前端了！！！！
